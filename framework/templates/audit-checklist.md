# Audit Checklist

Use this checklist for every scheduled project or portfolio audit.

## 1. Backlog Health

- [ ] Backlog exists and is actively maintained.
- [ ] New work is entering through backlog before requirement creation.
- [ ] Backlog items have status, priority, assignee, value statement, and next action.

## 2. Requirement Governance

- [ ] New requirements passed the requirement checklist.
- [ ] Requirements include calculated business value metrics.
- [ ] Requirements are traceable to backlog items where applicable.

## 3. Decision Governance

- [ ] New decisions passed the decision checklist.
- [ ] Decision register is current.
- [ ] Decision conflicts and triage were reviewed.

## 4. Technology Stack Governance

- [ ] Technology stack document exists.
- [ ] Major stack choices are linked to decision records.
- [ ] Solutions and implementation plans follow the approved stack.

## 5. Delivery Traceability

- [ ] Traceability matrix is current.
- [ ] Requirements are linked to solutions, decisions, implementation, testing, and release artifacts.
- [ ] Closed work is reflected accurately in project records.

## 6. Testing and Release Discipline

- [ ] Test plans exist for active delivery work.
- [ ] Release records exist for deployed changes.
- [ ] Follow-up actions from prior releases are tracked.

## 7. Portfolio Alignment

- [ ] Project roadmap is current.
- [ ] Cross-project dependencies are recorded when relevant.
- [ ] Project status matches portfolio reporting.

## 8. Audit Outcome

- [ ] Findings are recorded in the audit log.
- [ ] Follow-up actions have owners.
- [ ] Next audit date is scheduled.

## Result

Mark the audit:

- `Pass` if governance artifacts are current and traceable.
- `Warning` if gaps exist but do not block governance.
- `Fail` if major controls are missing or unreliable.
