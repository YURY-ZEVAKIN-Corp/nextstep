# Project Charter

## Metadata

- Project Key: `CQM`
- Project Name: `Customer Queue Manager`
- Owner: `TBD`
- Status: `Draft`
- Start Date: `2026-05-17`
- Target Outcome: `Create a system for handling electronic queues that helps customers wait well.`

## Purpose

Build a customer queue management system that provides clear wait-time visibility, fair queue handling, and better waiting experience.

## Success Measures

- Customers can see their place in queue and estimated wait time.
- Staff can manage queue flow and call the next customer reliably.
- Reduced complaints related to unclear waiting order and delays.

## Scope

- Electronic queue ticket creation and tracking.
- Customer-facing queue status view.
- Staff-facing queue management controls.
- Basic notifications when a customer is near service.

## Out of Scope

- Payment processing.
- Full CRM replacement.

## Key Stakeholders

- Customers
- Worker
- Operations manager

## Dependencies

- Display channel (screen, web, or mobile)
- Notification provider (SMS, email, or push)

## Risks

- Incorrect ETA calculations can reduce trust.
- Peak-hour scaling issues may affect queue updates.
