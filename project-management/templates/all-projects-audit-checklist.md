# All Projects High-Level Audit Checklist

Use this checklist for portfolio-level reviews across all active projects.

## 1. Project Coverage

- [ ] All active projects are listed in the project register.
- [ ] All active projects were included in the audit scope.
- [ ] Audit exceptions are documented.

## 2. Governance Compliance

- [ ] Each project has a backlog.
- [ ] Each project has requirement governance in place.
- [ ] Each project has decision governance in place.
- [ ] Each project has a technology stack document.
- [ ] Each project has a traceability matrix.

## 3. Delivery Health

- [ ] Project roadmaps are current.
- [ ] Active delivery work is traceable.
- [ ] Testing and release records exist where applicable.

## 4. Decision and Technology Risk

- [ ] Critical technology choices are backed by decision records.
- [ ] Cross-project conflicts or duplicated decisions are identified.
- [ ] Shared standards are being followed consistently.

## 5. Portfolio Alignment

- [ ] Project priorities align with the portfolio roadmap.
- [ ] Cross-project dependencies are visible.
- [ ] Blocked or high-risk projects are identified.

## 6. Audit Outcome

- [ ] Findings are summarized at portfolio level.
- [ ] Corrective actions are assigned.
- [ ] Next high-level audit date is scheduled.

## Result

Mark the audit:

- `Pass` if all active projects meet the expected governance baseline.
- `Warning` if some projects have gaps but portfolio control remains workable.
- `Fail` if governance is materially incomplete across the portfolio.
