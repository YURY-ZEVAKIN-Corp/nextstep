## Purpose
Assess whether the framework itself remains effective, adopted, and fit for current portfolio needs.

## Inputs
- Portfolio-level artifacts
- Framework audit schedule
- Previous framework audit findings

## Preconditions
- Portfolio has active projects with audit history.

## Steps
1. Execute framework audit checklist.
2. Sample projects for policy adherence and usability.
3. Identify control gaps, over-process, and missing guidance.
4. Propose framework-level improvements.
5. Record decisions and update standards/templates as approved.

## Outputs
- Framework audit record in `portfolio/framework-audit-log.md`
- Improvement backlog for framework changes

## Quality Checks
- Findings include cross-project evidence.
- Recommendations are actionable and prioritized.
- Ownership and target dates are assigned.

## Status Transition
- N/A (governance activity)

## RACI
- R: Portfolio Governance Function
- A: Portfolio Manager / PMO Lead
- C: Project Managers, Architects, Analysts, QA Leads
- I: All project teams

## Tools/Templates
- `framework/templates/framework-audit-checklist.md`
- `runtime/portfolio/framework-audit-log.md`
- `runtime/portfolio/standards.md`

## Escalation
If systemic governance risk is high, escalate to executive sponsor for immediate remediation plan.
