# Portfolio Standards

Apply these standards to every project:

- Every project must have a unique project key.
- Every project must maintain a backlog as the intake source for work.
- Every project must maintain a sprint plan for iterative delivery tracking.
- Every backlog item must have a named team-member assignee.
- Every backlog item must include a target sprint or be marked `Unscheduled`.
- Every backlog item must satisfy the Definition of Ready before it can move to `Ready`.
- Team members may work across many projects.
- Every requirement must pass the requirement checklist before becoming `Ready`.
- Every requirement must include a calculated business value metric.
- Every project must have a technology stack document.
- Every major technology choice must be justified by a decision record.
- Every major technology choice must be checked against the Technology Radar and documented.
- Every decision must pass the decision checklist before it is accepted.
- Every project must maintain a decision register, and shared decisions must be listed in the portfolio decision register.
- Every project must follow a scheduled audit cadence.
- Every audit must use the standard audit checklist.
- The portfolio must periodically run a high-level audit across all active projects.
- The portfolio must periodically run a framework audit on the governance model itself.
- Every project must maintain its own traceability matrix.
- Shared dependencies must be recorded in the portfolio dependency log.
